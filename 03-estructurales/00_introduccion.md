# Patrones Estructurales

## ¿Qué son los Patrones Estructurales?

Los patrones estructurales se ocupan de cómo las clases y objetos se componen para formar estructuras más grandes. Estos patrones facilitan el diseño identificando formas simples de realizar relaciones entre entidades, ayudando a garantizar que cuando una parte del sistema cambia, la estructura completa no necesita cambiar.

## Propósito

El objetivo principal de los patrones estructurales es simplificar el diseño mediante la identificación de relaciones simples entre clases y objetos. Se centran en cómo las clases heredan de otras clases y cómo se componen de otras clases, proporcionando diferentes formas de crear relaciones entre objetos.

## Problemas que Resuelven

Los patrones estructurales ayudan a resolver problemas como:

- **Incompatibilidad de interfaces**: Cuando necesitas que clases con interfaces incompatibles trabajen juntas
- **Complejidad de subsistemas**: Cuando necesitas simplificar la interacción con sistemas complejos
- **Funcionalidad adicional**: Cuando necesitas agregar comportamiento a objetos sin modificar su código
- **Composición flexible**: Cuando necesitas estructuras de objetos flexibles y dinámicas

## Ventajas Principales

1. **Flexibilidad**: Facilitan cambios en la estructura sin afectar otras partes
2. **Reutilización**: Promueven la reutilización mediante la composición
3. **Simplicidad**: Simplifican las relaciones entre objetos
4. **Desacoplamiento**: Reducen las dependencias entre componentes


## Cuándo Usar Patrones Estructurales

Considera usar patrones estructurales cuando:

- Necesitas adaptar interfaces incompatibles
- Quieres añadir responsabilidades a objetos de forma dinámica
- Necesitas simplificar la interfaz de un subsistema complejo
- Quieres componer objetos en estructuras de árbol
- Necesitas desacoplar la abstracción de su implementación
